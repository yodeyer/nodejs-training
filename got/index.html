Client HTTP
La librairie got

Node.js fournit un client HTTP qui est assez bas-niveau. Il est donc préférable d'utiliser une librairie comme got qui ajoute notamment les fonctionnalités suivantes :

  • gestion du http et du https avec le même module ;
  • envoi et réception de JSON ;
  • gestion des redirections ;
  • utilisation de promesses.

Pour faire une requête GET, il suffit de passer l'URL en premier paramètre de got 

const got = require('got');

const { body } = await got('http://www.google.com');

  • response : un objet IncomingMessage qui représente la réponse HTTP (statusCode, headers...)
  • body : le body de la réponse (String, ou objet JSON en fonction des options passées à got)

Pour faire des requêtes plus complexes, il est aussi possible de passer un objet en deuxième paramètre : 

const res = await got('http://www.google.com/search', {
  method: 'GET',
  query: {
    q: 'request'
  }
});

La liste complète des options est disponible dans le readme. Les plus souvent utilisées sont :

  • method : la méthode HTTP (GET, POST...) ;
  • query : les paramètres en query-string ;
  • body : le corps de la requête ;
  • json: true : permet de spécifier que le body envoyé est en JSON, et permet de parser automatiquement la réponse ;
  • agent : permet de fournir un agent HTTP (utile notamment pour utiliser un proxy).

info
La librairie request est actuellement la plus utilisée comme client HTTP. Cependant, elle est dorénavant en maintenance et n'aura plus de nouvelles fonctionnalités.

Dans un nouveau module app/jokes.js :

  1. créer la fonction asynchrone getJoke() qui appelle l'API http://api.icndb.com/jokes/random et qui retourne le champ joke de la réponse ;
  2. tester getJoke() (utiliser nock pour bouchonner l'API) ;
  3. modifier getJoke() pour remplacer de la prénom de la joke par votre prénom (voir la documentation de l'API).

warning
  • Si vous utilisez un proxy, utilisez caw comme agent HTTP (il gère automatiquement le proxy en fonction des variables d'environnement ;
  • got va en cas d'erreur automatiquement réessayer de faire la requête (voir le paramètre retry) ;
  • Pensez également à configurer un timeout.
