Rappel JavaScript

Une bonne connaissance du langage JavaScript est indispensable pour aborder sereinement l'apprentissage du Node.js.

Cette partie contient les basiques du langage et les évolutions apportées par ES6.

Types et variables

JavaScript est un langage dont le typage est faible et dynamique. Cela signifie qu'il n'est pas nécessaire de déclarer le type d'une variable avant de l'utiliser. Le type de la variable sera automatiquement déterminé lorsque le programme sera exécuté. Cela signifie également que la même variable pourra avoir différents types au cours de son existence :

            
  const length = 16;           // Number
  const lastName = "Johnson";  // String
  const x = {                  // Object
    firstName:"John",
    lastName:"Doe"
  };
Les types sont dynamiques en JavaScript. Ce qui signifie qu'une même variable peut contenir des types de données différents, par exemple : 
            
  let x;           // Ici x est undefined
  x = 5;           // Ici x est un Number
  x = "John";      // Ici x est un String

Les types de données

Le dernier standard ECMAScript définit 7 types de données :

  • Les types de données primitifs : Boolean, Null, Undefined, Number, String et Symbol ;
  • et le type Object
            
  let isHolder = true;             // Type Boolean    - true / false
  let name = "John";               // Type String     - il est également possible d'utiliser les guillements simple : 'John'
  let size = 5;                    // Type Number     - les nombres flottants à précision double : de -(253 -1) et 253 -1
                                   //                   avec également +Infinity, -Infinity, et NaN
  let myPrivateMethod = Symbol();  // Type Symbol
  let x = null;                    // Type Null
  let x;                           // Type Undefined

La langage propose également des objets globaux standards héritant d'Object avec des syntaxes litérales, comme par exemple :

            
  // Les tableaux : Array
  const fruits = ['apples', 'oranges', 'lemons'];
  // Les expressions régulières : RegExp
  const textFiles = /\.(rdf|txt)$/;
  // Les dates : Date
  const now = new Date();
  // Les erreurs : Error
  const err = new Error('did i do that?');
  // Les fonctions : Function
  const sayHi = function() { return 'Hi' };

Ces classes sont typées comme des Object, elles héritent des méthodes comme : toString, valueOf, hashCode.

info
La liste complète de ces "objets globaux" est disponible sur la documentation du langage, voir Mozilla Developer Network - Objets globaux

Les variables

Il existe trois mots clés en JavaScript pour déclarer une variable : var, let et const.

  • const permet de déclarer une variable à assignation unique bindée lexicalement, ce qui veut dire qu'une variable déclarée avec const ne contiendra qu'une valeur qui sera scopée au niveau du bloc : 
                    
  function fn() {
    const foo = "bar"
    if (true) {
      const foo // SyntaxError, la variable a besoin d'être assignée
      const foo = "qux"
      foo = "norf" // SyntaxError, la variable ne peut pas être réassignée
      console.log(foo)
      // "qux", la variable appartient au scope de son bloc (le "if")
    }
    console.log(foo)
    // "bar", la variable appartient au scope de la fonction "fn"
  }
    warning
    const définit une constante au niveau de la référence uniquement, il est possible de modifier le contenu d'un tableau ou d'un objet : 
                      
  const students = [];
  students.push('John'); // est valide !
  • let est similaire au const sans la contrainte sur l'assignation. let est donc ré-assignable et scopé au bloc courant : 
                    
  function fn() {
    let foo = "bar";
    if (true) {
      let foo; // pas d'erreur, foo === undefined
      foo = "qux";
      console.log(foo);
      // "qux", la variable appartient au scope de son blocs (le "if")
    }
    console.log(foo);
    // "bar", la variable appartient au scope de son bloc (la fonction "fn")
  }
  • var permet de déclarer une variable scopée au niveau de la fonction : 
                    
  function fn() {
    try {
      console.log(x); // undefined dû au hoisting (au lieu de ReferenceError pour let ou const)
      var x = foo(); // la déclaration de x est valable pour toute la fonction "fn"
    } catch(err) {}
    bar(x); // ici x est bien initialisé avec la valeur de retour de foo()
  }
    warning
    Le scope au niveau de la fonction et le mécanisme de hoisting ne sont pas naturels, nous vous conseillons d'utiliser const et let dans 99.9% des cas !
Fonctions

Les fonctions font partie des briques fondamentales de JavaScript. Une fonction est une procédure JavaScript, un ensemble d'instructions effectuant une tâche ou calculant une valeur.


En JavaScript une fonction est définie par le mot clé function suivi d'un nom et de paranthèse ()

            
  function name(parameter1, parameter2, parameter3) {
    // code to be executed
  }

Le nombre d'argument d'une fonction est dynamique, arguments est un pseudo-tableau contenant les paramètres effectifs au moment de l'appel.

info
En ES6, il est préférable d'utiliser la syntaxe Mozilla Developer Network - Rest parameters qui permet de représenter un nombre indéfini d'arguments sous forme de tableau : function sum(...theArgs) {}

En JavaScript, par défaut, la valeur des paramètres d'une fonction sera undefined. Malgré tout, il peut être assez utile de pouvoir définir d'autres valeurs par défaut.

            
  function multiply(a, b = 1) {
    return a * b;
  }

  multiply(5, 2); // 10
  multiply(5);    // 5 - ici b est undefined, il prend la valeur de 1

Il est également possible de définir une fonction grâce aux expressions de fonction, ces fonctions peuvent ainsi être anonymes :

            
  let square = function (nb) { return nb * nb };
  let x = square(4); // x reçoit la valeur 16

Cette syntaxe permet de passer une fonction comme argument d'une autre fonction ou de définir une fonction de manière conditionnelle.

Les fonctions fléchées

Une expression de fonction fléchée (arrow function en anglais) permet d’avoir une syntaxe plus courte que les expressions de fonction et ne possède pas ses propres valeurs pour this, arguments, super, ou new.target. Les fonctions fléchées sont souvent anonymes et ne sont pas destinées à être utilisées pour déclarer des méthodes.

            
  const square = nb => nb * nb; // le return est implicite sur une seule ligne
  console.log(square(2)); // => 4

  const sum = (a, b) => { // les parathèses sont obligatoires si vous avez au moins deux arguments
    return a + b;
  }
  console.log(sum(2, 3)); => 5
info
Pour aller plus loin, voir Mozilla Developer Network - Fonctions
Structures itératives

JavaScript propose plusieurs structures itératives sur les tableaux et les objets.


Parcourir un tableau

            
  // le for "classique" pour les tableaux
  const count = [ 'one', 'two', 'three' ];
  for (const i = 0; i < count.length; i++) {
    console.log(i, count[i]);
  }

  // le forEach en mode fonctionel
  ['one', 'two', 'three'].forEach(elem => {
    console.log(elem);
  });

Quelques méthodes supplémentaires utiles pour les tableaux : map, filter :

            
  const numbers = [1, 2, 3, 4];

  // map() crée un nouveau tableau avec les résultats de l'appel d'une fonction fournie sur chaque élément du tableau
  const squareNumbers = numbers.map(elem => elem * elem);
  console.log(squareNumbers); // [1, 4, 9, 16]

  // filter() crée un nouveau tableau contenant tous les éléments du tableau d'origine qui remplissent une condition déterminée
  const evenNumbers = numbers.filter(elem => elem % 2 === 0);
  console.log(evenNumbers); // [2, 4]

Parcourir un objet

            
  // le for in pour les objets
  const person = { name: 'Jane', age: 28 };
  for (const attr in person) {
    console.log(attr, person[attr]);
  }

  // en utilisant Object.entries()
  console.log(Object.entries(person)); // [["name", "Jane"], ["age", 28]]
  // en utilisant Object.keys()
  Object.keys(person).forEach(key => {
    console.log(key, person[key]);
  });

L'instruction for...of permet de créer une boucle Array qui parcourt un objet itérable (ce qui inclut les objets Array, Map, Set, String, TypedArray, l'objet arguments, etc.) et qui permet d'exécuter une ou plusieurs instructions pour la valeur de chaque propriété.

            
  const m = new Map([["foo", "hello"], ["bar", "world"]]);
  for (const [name, value] of m) {
    console.log(name + "->" + value); //"foo->hello", "bar->world"
  }

  const obj = { foo: "hello", bar: "world" };
  for (const key of Object.keys(obj)) {
    console.log(key + "->" + obj[key]); // "foo->hello", "bar->world"
  }
info
D'autres opérations sont possibles sur les objets et les tableaux, voir Mozilla Developer Network - Object, Array
Structures conditionnelles

Comme beaucoup de langages inspirés du C, Javascript utilise le short-circuit evaluation pour les conditions :

Le if / else

            
  if(name === 'John') {
    ...
  } else if (name === 'Jane') {
    ...
  } else {
    ...
  }

Le switch case

            
  switch (foo()) {
    case 'orange':
    case 'apple':
      console.log('fruit');
    break;
    default:
      console.log('unknown');
  }

L'opérateur ternaire

L'opérateur ternaire est utilisable dans une expression à la différence du if else qui est une instruction.

            
  const passed = foo() ? ':+1:' : ':-1:';
Les classes

En JavaScript, les classes sont des fonctions spéciales. Il est possible de les déclarer de la même façon que les fonctions : par déclaration, ou par expression.
Comme sur la plupart des langages, il est possible de définir un ou plusieurs constructeurs, méthodes de prototype, methodes statiques de classe. 

            
  class Point {
    // Constructeur de la classe
    constructor(x, y) {
      this.x = x;
      this.y = y;
    }

    // Méthode de prototype
    getPosition() {
      return `This is a point with x: ${this.x}, y: ${this.y}`;
    }

    // Il est également possible de lier une fonction à une propriété
    get position() {
      return `This is a point with x: ${this.x}, y: ${this.y}`;
    }

    // Méthode statique de classe
    static distance(a, b) {
      const dx = a.x - b.x;
      const dy = a.y - b.y;
      return Math.hypot(dx, dy);
    }
  }

  const p1 = new Point(5, 5);
  const p2 = new Point(10, 10);

  console.log(p1.getPosition()); // "This is a point with x: 5, y: 5", getPosition() est une fonction
  console.log(p1.position); // "This is a point with x: 5, y: 5", position est une propriété
  console.log(Point.distance(p1, p2)); // 7.0710678118654755
info
Il est également possible de définir des propriétés de classe et de déclarer des champs privés et publiques (fonctionalité expérimentale). Plus d'information sur Mozilla Developer Network - Classes

L'héritage de classe

Le mot-clé extends, utilisé dans les déclarations ou les expressions de classes, permet de créer une classe qui hérite d'une autre classe (on parle aussi de "sous-classe" ou de "classe-fille").

            
  class Animal {
    constructor(name) {
      this.name = name;
    }

    speak() {
      console.log(this.name + ' speaks');
    }
  }

  class Dog extends Animal {
    constructor(name) {
      super(name); // appelle le constructeur parent avec le paramètre
    }
    speak() {
      console.log(this.name + ' barks.');
    }
  }
Astuce ES6

la spécification ES6 (ou ES2015) a apporté de nombreuses fonctionalités au langage.

Littéraux de gabarits

Les littéraux de gabarits ` ` sont des littéraux de chaînes de caractères permettant d'intégrer des expressions. Avec eux, on peut utiliser des chaînes de caractères multi-lignes et des fonctionnalités d'interpolation.

            
  const a = 5;
  const b = 10;
  console.log(`${a} + ${b} equals
  ${a+b}.`);
  // "5 + 10 equals
  // 15."
info
Il est également possible d'imbriquer des gabarits pour obtenir des chaînes de caractères configurables. Plus d'information sur Mozilla Developer Network - Littéraux de gabarits

Syntaxe de décomposition

La syntaxe de décomposition ... (spread operator en anglais) permet d'étendre un itérable (par exemple une expression de tableau ou une chaîne de caractères) en lieu et place de plusieurs arguments (pour les appels de fonctions) ou de plusieurs éléments (pour les littéraux de tableaux) ou de paires clés-valeurs (pour les littéraux d'objets).

            
  // Améliorer les littéraux de tableau
  const articulations = ['épaules', 'genoux'];
  const corps = ['têtes', ...articulations, 'bras', 'pieds'];
  // ["têtes", "épaules", "genoux", "bras", "pieds"]

  // Copier un tableau
  const arr = [1, 2, 3];
  const arr2 = [...arr];

  // Concaténer deux tableaux
  arr = [...arr, ...arr2];
info
La proposition relative à la décomposition des propriétés (actuellement au stade de proposition de niveau 4) vise à ajouter la décomposition des propriétés pour les littéraux objets. Plus d'information sur Mozilla Developer Network - Syntaxe de décomposition

Affectation par décomposition

L'affectation par décomposition (destructuring en anglais) est une expression JavaScript qui permet d'extraire (unpack en anglais) des données d'un tableau ou d'un objet grâce à une syntaxe dont la forme ressemble à la structure du tableau ou de l'objet.

            
// Destructuring d'un tableau avec des valeurs par défaut
var [a = 5, b = 7] = [1];
console.log(a); // 1
console.log(b); // 7

// Destructuring d'un objet avec des valeurs par défaut
var {a, b = 21} = {a:45};
console.log(a); // 45
console.log(b); // 21

var {a, b} = null;
console.log(a); // "TypeError: Cannot destructure property `a` of 'undefined' or 'null'.
info
Plus d'information et d'exemples sont disponibles sur Mozilla Developer Network - Affecter par décomposition
