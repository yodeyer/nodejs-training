Les méthodes _.get, _.set, _.has sur un objet permettent d'éviter les ReferenceError

Sinon.js

Sinon.js est une librairie qui propose notamment des fakes, et mocks pour JavaScript.

fake
Les Fakes ont été introduits avec Sinon v5. Ils simplifient et fusionnent les concepts de Spies et Stubs.

Un fake est une fonction qui enregistre ses arguments et retourne une valeur.

Ces fake peuvent être utilisés avec la fonction sinon.replace pour simuler le comportement d'un objet existant :


mock
Les Mocks ne modifient pas un objet mais créent un objet mock avec de fausses méthodes et des comportements attendus pré-définis.

Par exemple, si l'on souhaite s'assurer que la méthode smsManager.sendTo est bien appelée deux fois dans la fonction api() : 
            
const sinon = require('sinon');

const api() => {
smsManager.sendTo('Yann', 'Yo!');
smsManager.sendTo('Quentin', 'Yo!');
};

// On utilise un mock sur l'objet "smsManager"
const mock = sinon.mock(smsManager);
mock.expects('sendTo').twice();

// L'appel à la fonction "api" n'enverra pas les SMS
api();

// Propagera une exception s'il n'y a pas exactement 2 appels à "sendTo" dans "api()"
mock.verify();

info
Sinon apporte également d'autres fonctionnalités comme les stubs, spies, spy calls, fake timers, pour plus d'informations voir la documentation officielle.
